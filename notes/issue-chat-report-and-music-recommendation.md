## 목표

- 사용자는 AI 마음대화 채팅방에서 `오늘 대화 마무리`를 눌러 오늘의 대화를 정리한 마음 리포트를 받을 수 있다.
- 리포트는 생성 완료 시 사용자 선택 없이 무조건 저장된다.
- 리포트에는 대화 요약, 마음 리포트 구성 항목, 추천 노래가 포함된다.
- `작은 다음 행동` 기능은 이번 범위에서 제외한다.
- 대화가 충분하지 않은 경우 AI가 유저 마음을 과하게 추론하지 않도록, 리포트 생성 전 명확한 안내 모달을 제공한다.

## 사용자 경험

### 진입

- 채팅방 화면에 `오늘 대화 마무리` 버튼을 제공한다.
- 버튼 위치는 채팅 경험을 방해하지 않는 곳에 둔다.
  - 후보: 채팅방 상단 액션 영역, 또는 입력창 근처 보조 액션
- 버튼 문구는 종료 느낌이 너무 강하지 않도록 `오늘 대화 마무리`를 기본으로 한다.

### 충분한 대화가 있는 경우

- 사용자가 `오늘 대화 마무리`를 누르면 바로 리포트 생성 로딩 상태로 진입한다.
- 생성 완료 후 리포트 화면을 보여준다.
- 리포트는 자동 저장된다.
- 저장 완료 상태를 화면에 명시한다.

### 대화가 짧은 경우

- 사용자가 `오늘 대화 마무리`를 누르면 먼저 안내 모달을 띄운다.
- 안내 문구 예시:

```text
아직 제대로 된 마음 리포트를 작성하기에는 대화가 조금 짧습니다.
지금 리포트를 만들면, 확인된 내용만 바탕으로 간단히 정리됩니다.
```

- 모달 버튼:
  - `대화 더 이어가기`: 모달을 닫고 채팅방으로 돌아간다.
  - `그래도 리포트 만들기`: 짧은 대화 기반 리포트를 생성하고 자동 저장한다.
- 짧은 대화 리포트도 충분한 대화 리포트와 동일한 화면 템플릿을 사용한다.
- 단, 항목 내용은 단정하지 않고 확인 가능한 수준으로만 표시한다.

### 리포트 화면 구성

- 리포트 타입/신뢰도 표시
  - `대화 기반 리포트`
  - `짧은 대화 기반 리포트`
- 오늘의 대화 요약
- 대표 감정
- 감정 강도
- 주요 원인
- 마음 흐름
- 오늘의 문장
- 추천 노래

### 짧은 대화 리포트 내용 예시

- 오늘의 대화 요약:
  - `오늘은 짧게 대화를 시작했지만, 마음 상태나 상황을 파악할 만큼의 내용은 충분히 쌓이지 않았습니다.`
- 대표 감정:
  - `아직 판단하기 어려움`
- 감정 강도:
  - `아직 판단하기 어려움`
- 주요 원인:
  - `확인되지 않음`
- 마음 흐름:
  - `대화량이 적어 마음의 변화 흐름을 정리하기 어렵습니다.`
- 오늘의 문장:
  - `아직 더 말해도 괜찮습니다.`
- 추천 노래:
  - 감정 단정 기반 추천이 아니라, 가볍게 마음을 정리하기 좋은 곡으로 추천한다.

### 추천 노래

- 리포트 안에 추천 노래 3곡을 제공한다.
- 각 노래는 다음 정보를 가진다.
  - 곡명
  - 아티스트
  - 추천 이유
  - YouTube 링크
- MVP에서는 자체 플레이어를 제공하지 않는다.
- YouTube 링크는 검색 링크로 제공한다.
  - 예: `https://www.youtube.com/results?search_query=아이유+밤편지`
- 추후 고도화 시 관리자가 검수한 공식 영상 URL을 저장할 수 있다.

## API Contract

### 대화 마무리 가능 여부 확인

- method/path 후보: `GET /api/ai-chat/rooms/today/report-readiness`
- response 예시:

```json
{
  "ready": true,
  "reason": "SUFFICIENT_CONVERSATION",
  "userMessageCount": 5,
  "userTextLength": 320
}
```

- 짧은 대화 response 예시:

```json
{
  "ready": false,
  "reason": "SHORT_CONVERSATION",
  "userMessageCount": 1,
  "userTextLength": 12,
  "guideMessage": "아직 제대로 된 마음 리포트를 작성하기에는 대화가 조금 짧습니다."
}
```

### 리포트 생성 및 저장

- method/path 후보: `POST /api/ai-chat/rooms/today/reports`
- request 예시:

```json
{
  "forceCreate": false,
  "clientRequestId": "report-..."
}
```

- 짧은 대화 안내 후 `그래도 리포트 만들기`를 선택한 경우:

```json
{
  "forceCreate": true,
  "clientRequestId": "report-..."
}
```

- response 예시:

```json
{
  "reportId": 1,
  "reportType": "FULL",
  "conversationDate": "2026-06-12",
  "summary": "오늘은 업무 부담과 회복 부족에 대한 이야기가 주로 오갔습니다.",
  "primaryEmotion": "지침",
  "emotionIntensity": 4,
  "mainCause": "업무 압박",
  "emotionalFlow": "초반에는 답답함이 컸고, 후반에는 상황을 정리하고 싶어 하는 마음이 드러났습니다.",
  "todaySentence": "지금은 더 밀어붙이기보다 회복 시간을 확보하는 것이 필요합니다.",
  "songs": [
    {
      "title": "밤편지",
      "artist": "아이유",
      "reason": "조용히 마음을 정리하고 싶은 밤에 어울립니다.",
      "youtubeUrl": "https://www.youtube.com/results?search_query=아이유+밤편지"
    }
  ],
  "saved": true,
  "createdAt": "2026-06-12T12:00:00+09:00"
}
```

### 리포트 조회

- method/path 후보: `GET /api/ai-chat/rooms/today/reports/latest`
- 오늘 저장된 리포트가 있으면 최신 리포트를 반환한다.
- 이미 오늘 리포트가 생성된 경우 중복 생성 정책을 정해야 한다.
  - MVP 추천: 오늘 리포트가 있으면 기존 리포트를 보여주고, `다시 생성`은 별도 후순위로 둔다.

## 백엔드 구현

- 오늘 대화방 기준으로 리포트 readiness를 판단한다.
- 충분한 대화 기준은 다음을 기본값으로 한다.
  - 유저 메시지 3개 이상
  - 유저 메시지 총 글자 수 80자 이상
  - 감정/상황/원인 중 최소 1개 이상 감지 가능
- 위 기준을 모두 만족하면 `FULL` 리포트를 생성한다.
- 기준 미달이고 `forceCreate=false`이면 리포트를 생성하지 않고 짧은 대화 안내 응답을 반환한다.
- 기준 미달이고 `forceCreate=true`이면 `SHORT` 리포트를 생성한다.
- `SHORT` 리포트에서는 감정, 강도, 원인을 단정하지 않는다.
- 리포트 생성 결과는 DB에 저장한다.
- 저장 테이블 후보:
  - `ai_chat_reports`
  - `ai_chat_report_songs`
- 리포트는 회원별, 날짜별, 대화방별로 조회 가능해야 한다.
- `clientRequestId`를 사용해 중복 생성 요청을 방지한다.
- OpenAI 또는 AI 응답 실패 시에도 사용자가 이해 가능한 오류 응답을 제공한다.
- 위기 감지/안전 안내와 리포트 생성은 혼동하지 않는다. 리포트는 진단 문구를 사용하지 않는다.

## 프론트엔드 구현

- 채팅방에 `오늘 대화 마무리` 액션을 추가한다.
- `오늘 대화 마무리` 클릭 시 readiness API를 호출한다.
- 충분한 대화이면 리포트 생성 로딩 화면 또는 모달을 보여준다.
- 짧은 대화이면 안내 모달을 보여준다.
- 안내 모달에서 `대화 더 이어가기`를 누르면 채팅방으로 복귀한다.
- 안내 모달에서 `그래도 리포트 만들기`를 누르면 `forceCreate=true`로 리포트 생성 API를 호출한다.
- 리포트 생성 완료 후 리포트 화면 또는 모달을 보여준다.
- 리포트 화면에는 `자동 저장됨` 상태를 명시한다.
- 추천 노래는 외부 링크로 열리게 한다.
- 링크는 새 탭으로 열고, 접근성 관점에서 외부 이동임을 알 수 있어야 한다.
- 모바일 화면에서도 리포트 항목과 노래 카드가 읽기 쉽게 배치되어야 한다.

## QA 기준

- 충분한 대화가 있는 상태에서 `오늘 대화 마무리`를 누르면 리포트가 바로 생성되고 저장되는가.
- 짧은 대화 상태에서 `오늘 대화 마무리`를 누르면 안내 모달이 먼저 뜨는가.
- 짧은 대화 안내 모달 문구가 유저에게 충분히 명확한가.
- `대화 더 이어가기`를 누르면 리포트가 생성되지 않고 채팅방으로 돌아가는가.
- `그래도 리포트 만들기`를 누르면 동일 리포트 템플릿으로 `SHORT` 리포트가 생성되고 저장되는가.
- `SHORT` 리포트에서 대표 감정, 감정 강도, 주요 원인을 억지로 단정하지 않는가.
- 생성된 리포트는 새로고침 후에도 조회되는가.
- 이미 오늘 리포트가 있는 경우 중복 생성 없이 기존 리포트를 보여주는가.
- 추천 노래의 YouTube 링크가 클릭 가능하고 새 탭으로 열리는가.
- 자체 음악 플레이어가 노출되지 않는가.
- 작은 다음 행동 항목이 리포트에 포함되지 않는가.
- API 실패 시 사용자가 이해 가능한 오류 상태를 보여주는가.
- 모바일 화면에서 모달, 리포트, 노래 카드가 잘리지 않는가.

## 완료 기준

- 채팅방에서 오늘 대화 마무리 플로우에 진입할 수 있다.
- 충분한 대화와 짧은 대화의 분기 UX가 구현되어 있다.
- 짧은 대화 안내 모달에 `대화 더 이어가기`, `그래도 리포트 만들기` 선택지가 있다.
- 충분한 대화 리포트와 짧은 대화 리포트가 동일한 화면 템플릿을 사용한다.
- 짧은 대화 리포트는 확인되지 않은 감정/원인을 단정하지 않는다.
- 리포트는 생성 후 무조건 저장된다.
- 리포트에는 대화 요약, 대표 감정, 감정 강도, 주요 원인, 마음 흐름, 오늘의 문장, 추천 노래가 포함된다.
- 리포트에는 작은 다음 행동이 포함되지 않는다.
- 추천 노래는 YouTube 링크로 이동할 수 있다.
- QA 기준을 통과한다.
