# 한국어 메시지 정책

StudyHub는 한국 사용자를 대상으로 하는 서비스다. 하네스가 생성하는 애플리케이션 코드와 사람이 읽는 하네스 리포트는 한국어를 우선한다.

## 애플리케이션 사용자 메시지

- StudyHub 사용자에게 보이는 모든 메시지는 한국어로 작성한다.
- 프론트엔드 검증 문구, 버튼 라벨, placeholder, 빈 상태, 성공 메시지, 오류 메시지는 한국어로 작성한다.
- 백엔드 API 오류 응답은 한국어이며, 안전하고, 사용자가 다음 행동을 이해할 수 있어야 한다.
- 내부 구현 정보, 클래스명, SQL 상세, stack trace, 외부 시스템 원문 오류, token, secret은 사용자에게 노출하지 않는다.

## 내부 예외와 로그

- 내부 예외 메시지와 로그는 한국어를 우선한다.
- 로그는 검색 가능하고 구조화되어야 한다.
- 실패 로그는 가능하면 아래 고정 key를 포함한다.
  - `who`
  - `what`
  - `requestData`
  - `reason`
- 민감 정보는 마스킹하거나 생략한다.
- 예상 가능한 비즈니스 검증 실패는 warn 수준 로그를 우선 사용한다.
- 시스템 장애나 stack trace가 필요한 실패는 error 수준 로그를 사용할 수 있다.

## 하네스 댓글과 리포트

- 하네스가 생성하는 GitHub 댓글은 한국어 제목과 한국어 설명을 사용한다.
- Discord/Google Chat 알림은 한국어 사용자에게 보고하는 문장으로 작성한다.
- 사람이 읽는 API 응답의 `reason`, `message` 필드는 한국어로 작성한다.
- `success`, `failed`, `ignored`, `needs_human` 같은 machine status 값은 안정적인 enum으로 유지하기 위해 영어를 사용할 수 있다.
